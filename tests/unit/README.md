# MoleditPy Unit Test Suite

This directory contains standalone unit tests for the core scientific logic and application modules of MoleditPy.

## Test Descriptions

### Unit Test Coverage & Logic Justification

| Test File | Component | Verification Scope |
| :--- | :--- | :--- |
| **`test_molecular_data.py`** | **Data Model (MolecularData)** | Verifies the core atom/bond storage and RDKit conversion.<br>• **CRUD Integrity**: ensures `add_atom`, `remove_atom`, and neighbor updates are consistent.<br>• **RDKit Round-trip**: Validates that converting to/from RDKit molecules preserves formal charges, radical counts, and coordinates. |
| **`test_string_importers.py`** | **SMILES/InChI Importers** | Verifies chemical parsing from strings.<br>• **Parsing Accuracy**: Checks atom/bond counts and molecular formulas against RDKit reference values.<br>• **Error Handling**: Ensures invalid or empty strings are handled gracefully. |
| **`test_app_state.py`** | **Application State Persistence** | Verifies the integrity of the application's internal state.<br>• **JSON Round-trip**: Ensures `get_current_state` and `set_state_from_data` preserve all molecular and UI data.<br>• **3D Persistence**: Confirms that optimized 3D structures and user constraints are correctly serialized. |
| **`test_plugin_manager.py`** | **Extensibility & Plugins** | Verifies the safe discovery and registration of plugins.<br>• **Discovery**: Tests that directory scanning ignores dunder files and correctly identifies packages.<br>• **Metadata Extraction**: Validates safe AST-based parsing of author, version, and name from plugin files. |
| **`test_plugin_interface.py`** | **Plugin Context & 3D Controller** | Verifies the plugin API surface.<br>• **PluginContext Methods**: Tests safe wrappers for atom access, selection, and molecule retrieval.<br>• **Plugin3DController**: Tests 3D view manipulation methods. |
| **`test_plugin_manager_window.py`** | **Plugin Manager GUI** | Verifies the graphical interactive components of the Plugin Manager.<br>• **Drag & Drop**: Tests installing plugins by dropping zip files, directories, and python scripts into the UI.<br>• **Action Handlers**: Validates logic for plugin deletion, dynamic status updates, and external navigation without blocking the main event loop. |
| **`test_plugin_manager_extended.py`** | **Plugin Backend Edge Cases** | Verifies highly specific error paths and system safeguards in the plugin architecture.<br>• **Robust Execution**: Ensures exceptions during plugin initialization or autorun do not crash the discovery loop.<br>• **Complex Metadata & IO**: Validates AST parsing for edge-case tuple versions and guards against OS-level I/O directory creation failures. |
| **`test_app_logic.py`** | **Core Application Orchestration** | Verifies the thread safety and state management of the application. Key scenarios include:<br>• **Async Worker Control**: Ensuring calculation threads correctly respond to halt signals.<br>• **Stereochemistry Persistence**: Confirming E/Z flags survive round-trip serialization even when backend sanitization might alter them. |
| **`test_hydrogen.py`** | **Chemical Intelligence** | Verifies the logic for implicit vs. explicit hydrogen management.<br>• **Smart Addition**: checks that hydrogens are added according to free valency rules.<br>• **Selective Removal**: Ensures only explicit hydrogens are removed while strictly preserving stereochemical configuration. |
| **`test_stereochemistry.py`** | **Stereo Flags & UI Mapping** | Verifies the translation layer between the internal RDKit representation and the UI's visual indicators.<br>• **Chiral Mapping**: Ensures atomic chirality is correctly parsed and visualized.<br>• **Stereo Inversion**: Confirmed via `test_stereo_confirmation`, confirming `MirrorDialog` correctly flips R/S centers during coordinate inversion.<br>• **Stereo Loss**: Confirms that `PlanarizeDialog` removes chiral markers when centers are flattened. |
| **`test_geometry.py`** | **2D/3D Geometrical Operations** | Verifies geometrical manipulations of molecular coordinates.<br>• **Coordinate Mirroring**: Checks algorithms for mirroring and inversion in `MirrorDialog`, ensuring R/S consistency.<br>• **Planarization Logic**: Confirmed via `test_planarize_logic`, confirming SVD-based best-fit plane projection maintains exact coplanarity (Z-variance = 0). |
| **`test_properties.py`** | **Chemical Properties Analysis** | Verifies the calculation and presentation of molecular descriptors.<br>• **Descriptor Accuracy**: Validates calculations like MW and LogP against known standards.<br>• **Error Stability**: Ensures the analysis window handles calculation failures gracefully without crashing the UI. |
| **`test_parsers_extended.py`** | **Unified File I/O & Parsing** | Verifies the robustness of data ingestion modules for MOL, SDF, and XYZ formats.<br>• **XYZ/MOL Loading**: Tests custom parsers designed to handle non-standard or slightly malformed files.<br>• **MOL Block Fixing**: Tests `fix_mol_counts_line` and `fix_mol_block` to ensure legacy MOL files missing V2000 tags are readable.<br>• **XYZ Edge Cases**: Verifies user prompts for charge estimation, skip-chemistry paths, and symbol capitalization coercion. |
| **`test_export_logic.py`** | **Multi-Format Export** | Verifies the integrity of scientific data exports.<br>• **OBJ/MTL Material Mapping**: Ensures molecular colors and names are correctly mapped to standard material libraries for 3D printing/rendering.<br>• **Vector/Mesh Processing**: Validates the merging and extraction of PolyData actors for headless export. |
| **`test_main_window_export_extended.py`** | **Extended Export Coverage** | Verifies additional export paths including STL, color STL, and 2D PNG/SVG export logic. |
| **`test_scene_interactions.py`** | **2D Interactive Behavior** | Verifies the visual feedback and data updates during sketching.<br>• **Chemical Property Toggles**: Ensures the persistent storage of radical and charge state through repetitive mouse clicks.<br>• **Fragment Fusion**: Validates the skeletal fusion of complex rings (e.g., benzene) and their interaction with existing bonds. |
| **`test_scene_extended.py`** | **Advanced Scene & Interaction Logic** | Verifies complex interaction patterns and advanced scene features.<br>• **Keyboard Shortcuts**: Systematically tests mode switching, deletion, and property toggling via keys.<br>• **Bond Logic**: Validates E/Z stereochemistry toggles and bond direction reversal.<br>• **Undo/Redo & Templates**: Consolidated tests for history management and custom fragment placement logic. |
| **`test_edit_actions.py`** | **Main Window Edit Actions** | Verifies 2D molecular manipulations.<br>• **2D Rotation**: ensures coordinates are correctly rotated around the selection center.<br>• **Overlap Resolution**: Validates the algorithm for resolving coincident atoms.<br>• **Asynchronous Updates**: Verifies `update_implicit_hydrogens` via `QTimer` mocking. |
| **`test_compute_logic.py`** | **Main Window Compute Logic** | Verifies the 2D-to-3D conversion workflow.<br>• **Trigger Conversion**: Tests edge cases including `DetectChemistryProblems`, `SanitizeMol` failures, and collision detection for multiple fragments.<br>• **Worker Management**: Ensures active workers are accurately tracked, stale IDs are ignored, and halt signals are correctly processed.<br>• **Format Compatibility**: Validates handling of both modern and legacy worker result/error payloads. |
| **`test_items_visual.py`** | **Graphics Items (Atom/Bond)** | Verifies the visual state and rendering logic of UI elements.<br>• **Charge/Radical Display**: Ensures visual markers update correctly on state change.<br>• **Ring Rendering**: Validates BondItem logic for ring-specific dash/solid styles. |
| **`test_atom_bond_items.py`** | **AtomItem & BondItem Properties** | Verifies AtomItem painting logic (radicals, selection, hover, problem highlights, implicit hydrogens) and BondItem ring double bond and E/Z label bounding box logic. |
| **`test_project_io_extended.py`** | **Project Serialization Logic** | Verifies the integrity of the application's native save format (`.pmeprj`).<br>• **State Persistence**: Ensures the generic JSON structure, canvas positions, and view states round-trip correctly. |
| **`test_calculation_worker_direct.py`** | **Async Worker Logic** | Verifies the core `CalculationWorker` state machine and thread management. |
| **`test_edit_3d_logic.py`** | **3D Geometrical Editing** | Verifies 3D-specific manipulations like direct atom dragging and coordinate updates. |
| **`test_main_window_init_coverage.py`** | **Initialization Stability** | Verifies the cold-start stability and registration of core application modules. |
| **`test_io.py`** | **Legacy Project IO** | (Consolidated into `test_project_io_extended.py`) |
| **`test_parsers.py`** | **Legacy Parsers** | (Consolidated into `test_parsers_extended.py`) |
| **`test_scene_advanced.py`** | **Legacy Scene Logic** | (Consolidated into `test_scene_extended.py`) |
| **`test_about_dialog.py`** | **About Dialog** | Verifies `AboutDialog` initialization and edge-case interactions.<br>• **Initialization**: Confirms title and image label are set correctly.<br>• **Easter Egg**: Validates right-click triggers the hidden `clear_all` + SMILES load sequence.<br>• **Input Guard**: Ensures left-clicks are silently ignored. |
| **`test_app_state_persistence.py`** | **Full Project Serialization** | Verifies complete round-trip serialization via the `.pmeprj` format.<br>• **State Roundtrip**: Confirms 2D/3D structures, constraints, and view state survive save/load cycles.<br>• **Binary Undo**: Validates binary-serialized undo/redo state is correctly preserved. |
| **`test_atom_placement_logic.py`** | **Atom Placement Geometry** | Verifies the bond-aware positioning algorithm for new atoms.<br>• **Neighbor Cases**: Tests placement geometry for 0–3 existing neighbors.<br>• **Fallback Logic**: Validates balanced vs. unbalanced geometry selection. |
| **`test_benzene_placement_shortcut.py`** | **Benzene Shortcut (Key_4)** | Verifies the `Key_4` shortcut behavior for single-click benzene placement.<br>• **Atom/Bond Target**: Confirms one-shot placement when hovering over an atom or bond.<br>• **Empty Space**: Validates mode-switch behavior when no target is under cursor. |
| **`test_benzene_rotation.py`** | **Template Fusion Rotation Scoring** | Verifies ring rotation scoring during template fusion onto existing structures.<br>• **Score Factors**: Tests single/multi-edge attachment and connection safety scoring.<br>• **Best Angle**: Confirms the highest-scoring rotation is selected. |
| **`test_calculation_worker_optimize.py`** | **Optimization Modes** | Verifies the `CalculationWorker` optimization pipeline across all force fields.<br>• **Force Fields**: Tests MMFF94s, UFF, and Open Babel optimization paths.<br>• **Collision Avoidance**: Validates that multi-fragment results trigger collision-resolution passes.<br>• **Halt Signal**: Ensures in-flight optimization stops cleanly on interrupt. |
| **`test_calculation_worker_optimize_intermolecular.py`** | **Intermolecular Interaction Toggle** | Verifies the `ignoreInterfragInteractions` flag behavior.<br>• **MMFF94s / UFF**: Confirms the flag is correctly passed to both force-field backends. |
| **`test_color_settings_dialog.py`** | **Color Settings Dialog** | Verifies `ColorSettingsDialog` UI and persistence.<br>• **Initialization**: Confirms color pickers populate from current settings.<br>• **Reset**: Validates reset button restores default colors.<br>• **Persistence**: Ensures accepted changes are written back to settings. |
| **`test_custom_interactor_style.py`** | **3D Interactor Style** | Verifies VTK interactor behavior in atom-edit mode.<br>• **Atom Selection**: Confirms left-click in edit mode performs a pick operation.<br>• **Background Click**: Validates trackball rotation is invoked when no atom is picked. |
| **`test_dialog_logic.py`** | **Dialog Manager Methods** | Comprehensive tests for `DialogManager` dispatch logic.<br>• **Standard Dialogs**: Tests about, periodic table, and analysis dialog launch paths.<br>• **Template Dialogs**: Verifies save/load of user-defined chemical fragments.<br>• **Geometry Dialogs**: Tests bond length, angle, dihedral, and alignment dialog wiring.<br>• **Modeless Lifecycle**: Validates signal-based teardown for non-blocking dialogs. |
| **`test_dialog_manager.py`** | **Dialog Manager Edge Cases** | Supplementary tests for `DialogManager` error paths and redundant-call guards. |
| **`test_edit_actions_extended.py`** | **Extended Edit Actions** | Verifies `EditActionsManager` coverage beyond the core suite.<br>• **Chemistry Checks**: Tests valency enforcement and pre-edit validation guards.<br>• **Clipboard**: Validates copy/paste with MIME serialization.<br>• **Collision Avoidance**: Tests the overlap-resolution pass on paste operations.<br>• **2D Rotation**: Verifies precise coordinate rotation around the selection centroid. |
| **`test_io_manager.py`** | **IO Manager** | Verifies `IOManager` file I/O workflows.<br>• **Charge Prompting**: Confirms XYZ imports request formal charge from user when ambiguous.<br>• **XYZ Load**: Tests the custom XYZ parser integration path.<br>• **3D MOL Save**: Validates header replacement when writing optimized 3D structures. |
| **`test_measurement_calcs.py`** | **3D Measurement & Transformation Dialogs** | Verifies calculation correctness for all geometry dialogs.<br>• **Internal Coordinates**: Tests bond length, angle, and dihedral value computation.<br>• **Transforms**: Validates translation, axis alignment, and plane alignment logic.<br>• **Move Group**: Confirms group-translation correctly shifts a subset of atoms. |
| **`test_parser_robustness.py`** | **Parser Robustness** | Verifies edge-case handling in file parsers and utilities.<br>• **MOL Count Line**: Tests `fix_mol_counts_line` for missing/malformed V2000 tags.<br>• **XYZ Robustness**: Validates recovery from unusual whitespace and symbol casing.<br>• **Safe Properties**: Confirms getter/setter utilities handle missing attributes gracefully. |
| **`test_project_io_raw.py`** | **Raw Project IO (.pmeraw)** | Verifies the pickle-based `.pmeraw` project format.<br>• **Save/Load**: Tests the full binary serialization round-trip.<br>• **Error Handling**: Confirms corrupt or missing files raise informative errors. |
| **`test_radical_toggle.py`** | **Radical Toggle** | Verifies radical electron cycling via the `Period` key shortcut.<br>• **Selected Atoms**: Tests the toggle increments radical count on all selected atoms.<br>• **Cursor Target**: Confirms the key also acts on the atom currently under the cursor. |
| **`test_scene_extended_logic.py`** | **Scene Extended Logic** | Verifies advanced 2D-scene event handlers.<br>• **E/Z Toggle**: Tests stereo-bond cycling on double bonds.<br>• **Bond Inversion**: Validates the direction-reversal action for wedge/dash bonds.<br>• **Template Preview**: Confirms benzene ghost-preview rotation selects the best angle. |
| **`test_slider_logic.py`** | **Angle Wrapping Logic** | Verifies angular normalization helpers used by geometry dialogs.<br>• **Angle Wrap**: Confirms values outside ±180° are clamped/wrapped correctly.<br>• **Dihedral Wrap**: Validates dihedral wrapping to the ±180° domain. |
| **`test_ui_manager_robustness.py`** | **UI Manager Robustness** | Verifies `UIManager` defensive behaviour.<br>• **Close Events**: Confirms the manager handles window-close without recursion.<br>• **3D Feature Toggle**: Tests enabling/disabling 3D-dependent toolbar actions.<br>• **Mode Switch**: Validates template-preview hiding during tool-mode transitions. |
| **`test_utils_sip.py`** | **SIP Safety Utility** | Verifies the `sip_isdeleted_safe` helper for PyQt object lifetime checks.<br>• **Deleted Objects**: Confirms the helper returns `True` for already-deleted C++ objects.<br>• **Exception Safety**: Validates that internal SIP errors are caught and surfaced cleanly. |
| **`test_view_3d.py`** | **3D View Rendering** | Verifies `View3DManager` rendering entry points.<br>• **Display Styles**: Tests that Ball & Stick, CPK, Wireframe, and Stick styles render without error.<br>• **None Molecule**: Confirms a `None` molecule input is handled gracefully without crash. |
| **`test_view_3d_logic_extended.py`** | **Extended 3D Visualization** | Verifies the detailed rendering pipeline in `View3DManager`.<br>• **Atom Glyphs**: Tests radius and material mapping for each CPK style.<br>• **Bond Cylinders**: Validates bond geometry generation.<br>• **Aromatic Rings**: Confirms torus representations for aromatic systems.<br>• **Double Bond Offsets**: Tests the offset computation for parallel double-bond lines.<br>• **Labels**: Verifies EZ stereo and chiral (R/S) label actor generation and positioning.<br>• **Color Overrides**: Confirms per-atom color overrides are applied correctly. |
| **`test_worker_robustness.py`** | **Worker Robustness** | Verifies `CalculationWorker` defensive paths.<br>• **Halt Logic**: Confirms the worker stops within one iteration cycle after a halt signal.<br>• **Iterative Optimization**: Tests that repeated optimization rounds converge and do not diverge on standard molecules.<br>• **Empty Input**: Validates that a molecule with no atoms raises a clear error rather than crashing. |

## Mocking Strategy
These tests use a `MockMainWindow` (defined in `conftest.py`) which simulates the host environment with standardized `CalculationWorker`, `MolecularData`, and logic implementations from the `core` and `ui` packages.
