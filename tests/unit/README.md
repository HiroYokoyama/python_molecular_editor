# MoleditPy Unit Test Suite

This directory contains standalone unit tests for the core scientific logic and application modules of MoleditPy.

## Test Descriptions

### Unit Test Coverage & Logic Justification

| Test File | Component | Verification Scope |
| :--- | :--- | :--- |
| **`test_molecular_data.py`** | **Data Model (MolecularData)** | Verifies the core atom/bond storage and RDKit conversion.<br>• **CRUD Integrity**: ensures `add_atom`, `remove_atom`, and neighbor updates are consistent.<br>• **RDKit Round-trip**: Validates that converting to/from RDKit molecules preserves formal charges, radical counts, and coordinates. |
| **`test_string_importers.py`** | **SMILES/InChI Importers** | Verifies chemical parsing from strings.<br>• **Parsing Accuracy**: Checks atom/bond counts and molecular formulas against RDKit reference values.<br>• **Error Handling**: Ensures invalid or empty strings are handled gracefully. |
| **`test_app_state.py`** | **Application State Persistence** | Verifies the integrity of the application's internal state.<br>• **JSON Round-trip**: Ensures `get_current_state` and `set_state_from_data` preserve all molecular and UI data.<br>• **3D Persistence**: Confirms that optimized 3D structures and user constraints are correctly serialized. |
| **`test_plugin_manager.py`** | **Extensibility & Plugins** | Verifies the safe discovery and registration of plugins.<br>• **Discovery**: Tests that directory scanning ignores dunder files and correctly identifies packages.<br>• **Metadata Extraction**: Validates safe AST-based parsing of author, version, and name from plugin files. |
| **`test_app_logic.py`** | **Core Application Orchestration** | Verifies the thread safety and state management of the application. Key scenarios include:<br>• **Async Worker Control**: Ensuring calculation threads correctly respond to halt signals.<br>• **Stereochemistry Persistence**: Confirming E/Z flags survive round-trip serialization even when backend sanitization might alter them.<br>• **Robust Fallbacks**: Testing custom `MolBlock` generation robustness when standard libraries fail on invalid valencies.<br>• **Coordinate Systems**: Validating the precise application of `ANGSTROM_PER_PIXEL` scaling during 2D-to-3D conversions. |
| **`test_hydrogen.py`** | **Chemical Intelligence** | Verifies the logic for implicit vs. explicit hydrogen management.<br>• **Smart Addition**: checks that hydrogens are added according to free valency rules.<br>• **Selective Removal**: Ensures only explicit hydrogens are removed while strictly preserving stereochemical configuration. |
| **`test_stereochemistry.py`** | **Stereo Flags & UI Mapping** | Verifies the translation layer between the internal RDKit representation and the UI's visual indicators.<br>• **Chiral Mapping**: Ensures atomic chirality is correctly parsed and visualized.<br>• **Stereo Inversion**: **Verified via `test_stereo_confirmation`**, confirming `MirrorDialog` correctly flips R/S centers during coordinate inversion.<br>• **Stereo Loss**: Confirms that `PlanarizeDialog` removes chiral markers when centers are flattened. |
| **`test_geometry.py`** | **2D/3D Geometrical Operations** | Verifies geometrical manipulations of molecular coordinates.<br>• **Coordinate Mirroring**: Checks algorithms for mirroring and inversion in `MirrorDialog`, ensuring R/S consistency.<br>• **Planarization Logic**: **Verified via `test_planarize_logic`**, confirming SVD-based best-fit plane projection maintains exact coplanarity (Z-variance = 0). |
| **`test_properties.py`** | **Chemical Properties Analysis** | Verifies the calculation and presentation of molecular descriptors.<br>• **Descriptor Accuracy**: Validates calculations like MW and LogP against known standards.<br>• **Error Stability**: Ensures the analysis window handles calculation failures gracefully without crashing the UI. |
| **`test_parsers.py`** | **File I/O & Parsing** | Verifies the robustness of data ingestion modules.<br>• **XYZ/MOL Loading**: Tests custom parsers designed to handle non-standard or slightly malformed files that strict standard libraries might reject. |
| **`test_parsers_extended.py`** | **Advanced Parsing Logic** | Verifies complex chemical data ingestion.<br>• **SDF/SDSuppliers**: Validates multi-record handling and fallback to `SDMolSupplier` for non-standard path extensions.<br>• **MOL Block Fixing**: Tests `fix_mol_counts_line` and `fix_mol_block` to ensure legacy MOL files missing V2000 tags are readable.<br>• **XYZ Edge Cases**: Verifies user prompts for charge estimation, skip-chemistry paths, and symbol capitalization coercion. |
| **`test_export_logic.py`** | **Multi-Format Export** | Verifies the integrity of scientific data exports.<br>• **OBJ/MTL Material Mapping**: Ensures molecular colors and names are correctly mapped to standard material libraries for 3D printing/rendering.<br>• **Vector/Mesh Processing**: Validates the merging and extraction of PolyData actors for headless export. |
| **`test_scene_interactions.py`** | **2D Interactive Behavior** | Verifies the visual feedback and data updates during sketching.<br>• **Chemical Property Toggles**: Ensures the persistent storage of radical and charge state through repetitive mouse clicks.<br>• **Fragment Fusion**: Validates the skeletal fusion of complex rings (e.g., benzene) and their interaction with existing bonds. |
| **`test_scene_extended.py`** | **Advanced Scene Logic** | Verifies complex interaction patterns.<br>• **Keyboard Shortcuts**: Systematically tests mode switching, deletion, and property toggling via keys.<br>• **Bond Logic**: Validates E/Z stereochemistry toggles and bond direction reversal.<br>• **User Templates**: Tests custom fragment placement and preview logic. |
| **`test_edit_actions.py`** | **Main Window Edit Actions** | Verifies 2D molecular manipulations.<br>• **2D Rotation**: ensures coordinates are correctly rotated around the selection center.<br>• **Overlap Resolution**: Validates the algorithm for resolving coincident atoms.<br>• **Asynchronous Updates**: Verifies `update_implicit_hydrogens` via `QTimer` mocking. |
| **`test_compute_logic.py`** | **Main Window Compute Logic** | Verifies the 2D-to-3D conversion workflow.<br>• **Trigger Conversion**: Tests edge cases including `DetectChemistryProblems`, `SanitizeMol` failures, and collision detection for multiple fragments.<br>• **Worker Management**: Ensures active workers are accurately tracked, stale IDs are ignored, and halt signals are correctly processed.<br>• **Format Compatibility**: Validates handling of both modern and legacy worker result/error payloads. |
| **`test_items_visual.py`** | **Graphics Items (Atom/Bond)** | Verifies the visual state and rendering logic of UI elements.<br>• **Charge/Radical Display**: Ensures visual markers update correctly on state change.<br>• **Ring Rendering**: Validates BondItem logic for ring-specific dash/solid styles. |
| **`test_io.py`** | **Project Serialization** | Verifies the integrity of the application's native save format.<br>• **.pmeprj Integrity**: Tests the generic JSON structure, ensuring all custom attributes (canvas positions, view states) round-trip correctly. |

## Mocking Strategy
These tests use a `MockMainWindow` (defined in `conftest.py`) which simulates the host environment with standardized `CalculationWorker`, `MolecularData`, and logic implementations from `main_window_app_state.py`.
